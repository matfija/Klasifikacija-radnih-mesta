Klasifikacija radnih mesta¶

U svesci je razmotrena primena tehnika obrade prirodnih jezika (natural language processing, NLP) na reavanje problema razvrstavanja radnih mesta (job title) prema senioritetu. Dostavljeni podaci su uitani i pripremljeni (normalizovani i vektorizovani) na vie razliitih naina, nakon ega je na delu radnih mesta obueno nekoliko klasifikacionih modela. Modeli su vrednovani prema ponaanju na drugom delu podataka, odnosno uspenosti predvianja senioriteta.

Pored samog koda i rezultata, usput su detaljno prokomentarisani svi preduzeti koraci. Posebno je zanimljivo napomenuti da je primenjeno poneto karakteristino za svaku od tri faze razvoja obrade prirodnih jezika kao oblasti: simboliki NLP (regularni izrazi, klasifikacija prema runo osmiljenim pravilima i slino), statistiki NLP (metode mainskog uenja), neuronski/duboki NLP (primena neuronskih mrea i dubokog uenja).

1. Učitavanje podataka ⮭¶

Za poetak, neophodno je uitati dostavljene podatke. Oni se nalaze u pet razliitih datoteka, od kojih su prve etiri (20.txt, 20-40.txt, 40-60.txt, 60-100.txt) planirane za pravljenje i evaluaciju modela, dok peta (test_titles.json) sadri neobeleena radna mesta i namenjena je proveri klasifikacije potpuno novih, nekategorizovanih i nezavisnih podataka. Sve nabrojane datoteke smetene su u direktorijum podaci.

In [1]:
# Podaci za pravljenje i evaluaciju modela
train_podaci = ['20.txt', '20-40.txt', '40-60.txt', '60-100.txt']

# Dodavanje direktorijuma na imena datoteka
for i, p in enumerate(train_podaci):
    train_podaci[i] = '../podaci/' + p

# Podaci za proveru klasifikacije na kraju

Deo podataka planiran za pravljenje i evaluaciju modela dat je u strukturiranom formatu sledeeg oblika.

{ name: 'chief financial officer', role: 'finance', levels: [ 'cxo' ], sub_role: null }

Na konkretnom primeru, naziv radnog mesta (name) jeste chief financial officer (finansijski direktor), uloga/odeljenje (role) jeste finance (finansije), nivo/senioritet (levels) jeste cxo (direktor), dok poduloga (sub_role) nije definisana. Na drugim primerima, postoji vie nivoa ili nisu definisani.

{ name: 'community teaching experience', role: null, levels: [], sub_role: null }

{ name: 'owner and lead programmer', role: 'engineering', levels: ['manager', 'owner'], sub_role: null }

U nastavku je napisana jednostavna klasa koja skladiti podatke prema navedenom ablonu. Moe i da ih poredi, kao i lepo da ih odtampa.

In [2]:
# Klasa koja opisuje radno mesto
class JobTitle:
    # Konstruktor koji samo pamti podatke
    def __init__(self, name, levels=None):
        self.name = name
        self.levels = levels
    
    # Poreenje (<) dva radna mesta
    def __lt__(self, other):
        return self.newname < other.newname\
               if hasattr(self, 'newname')\
               else self.name < other.name
    
    # Poreenje (==) dva radna mesta
    def __eq__(self, other):
        return self.newname == other.newname\
               if hasattr(self, 'newname')\
               else self.name == other.name
    
    # Predstavljanje objekta u vidu niske
    def __str__(self):
        return f'{{\n  name: {self.name},\n  levels: {self.levels}\n}}'

Ovaj format lii na poznati JSON, ali ipak ne koristi navodnike na pravi nain, tako da ga nije mogue uitati koristei ugraeni modul json. Umesto toga, iskorien je modul re, odnosno podaci su uitani koristei regularne izraze. Ovo je u duhu simbolikog NLP-ja, kao prve etape u razvoju ove oblasti.

In [3]:
# Modul za rad sa regularnim izrazima
import re

# Funkcija za uitavanje podataka iz datoteke
def uitaj(datoteka):
    # Otvaranje datoteke sa UTF-8 enkodiranjem
    with open(datoteka, encoding='utf-8') as f:
        # itanje celog teksta datoteke
        tekst = f.read()
    
    # Regularni izraz koji izdvaja jedno radno mesto
    regex = r'{\s+name: (.+?),\s+role: .+?,\s+levels: (.+?),\s+sub_role: .+?\s+}'
    
    # Vraanje svakog radnog mesta iz datoteke
    return [JobTitle(eval(name), eval(levels)) for
            name, levels in re.findall(regex, tekst)]

3. Normalizacija teksta ⮭¶

Normalizacija igra znaajnu ulogu u klasifikaciji teksta. Poeljno je sve nazive radnih mesta uiniti to uniformnijim. Prvi korak u tome jeste segmentacija teksta, to se u ovom sluaju svodi na izdvajanje rei i interpunkcije. Ovako izdvojene celine nazivaju se tokeni i mogu se dobiti pomou modula NLTK.

Dalje, dobro je da je svaka re svedena na neki osnovni oblik. Ovo je mogue uraditi stemovanjem, to je proces koji heuristiki pokuava da doe do osnove rei. Alternativno, moe se precizno pronai koren (lema, renika forma) rei procesom lematizacije. Oba su izvodljiva u uitanoj biblioteci NLTK. Vano je samo napomenuti da lematizator zahteva i vrstu rei kao argument, pa je za te potrebe iskorien ugraeni obeleiva vrste rei (POS tagger). Sam lematizator zasnovan je na spisku razliitih oblika rei prema WordNet-u, najpoznatijoj i najrazvijenijoj leksikoj bazi semantikih odnosa rei.

In [21]:
# Modul za rad sa naprednim renikom
from collections import defaultdict

# Renik koji je podrazumevano imenica
mapa = defaultdict(lambda: nltk.corpus.wordnet.NOUN)

# Popunjavanje ostalih skraenica
mapa['J'] = nltk.corpus.wordnet.ADJ
mapa['S'] = nltk.corpus.wordnet.ADJ_SAT
mapa['V'] = nltk.corpus.wordnet.VERB
mapa['R'] = nltk.corpus.wordnet.ADV

# Funkcija za tagovanje
tag = lambda j: mapa[j[0]]

# Instanciranje lematizatora
lema = nltk.stem.WordNetLemmatizer()

# Odreivanje vrste rei svakog radnog mesta
for p in podobel:
    pos = nltk.pos_tag(p.tokeni)
    
    # Lematizacija svakog tokena
    p.toklem = [lema.lemmatize(i, tag(j)) for i, j in pos]

# Modul sa posebnim niskama
import string

# Skidanje zalepljene interpunkcije
strip = lambda t: ''.join(c for c in t if c in string.ascii_lowercase)

# Provera da li je interpunkcija
def punkt(t):
    for c in t:
        if c not in string.punctuation:
            return False
    return True

# Provera da li su sve mala ASCII slova
def aski(t):
    for c in t:
        if c not in string.ascii_lowercase and c not in string.punctuation:
            return False
    return True

# Izbacivanje svega to nisu mala ASCII slova
for p in podobel:
    p.tokpunk = [strip(t) for t in p.toklem if not punkt(t) and aski(t)]

4. Klasifikaciona pravila ⮭¶

Imena su sada normalizovana. To, meutim, nije dovoljno kako bi bila prosleena klasifikacionim modelima na obuavanje. Problem je to modeli mahom ne prihvataju tekst (niske) ili nizove tokena, ve iskljuivo reprezentaciju u vidu vektora fiksne duine. To znai da je imena neophodno vektorizovati.

Postavlja se pitanje kako od teksta napraviti vektor. Ideja se moe pronai u simbolikom NLP-ju, kao prvoj etapi razvoja obrade prirodnih jezika. On zagovara upotrebu klasifikacionih pravila, koja direktno rade nad tekstom ili nizom tokena. Npr. moe se pretpostaviti da svako ime koje sadri re manager predstavlja jednog menadera (klasa manager), dok svaki potpredsednik (klasa vp) sadri re vice. Ovakva pravila pridruivanja mogu se i automatski pronai.

Uspenost pravila ocenjuje se dvema merama: odzivom, koji je udeo prepoznatih instanci ciljane klase, i preciznou, koja je udeo tanih prepoznavanja.

$$ \text{odziv} = \frac{|\text{dohvaeni ciljane klase}|}{|\text{svi ciljane klase}|} \; \; \; \; \; \; \text{preciznost} = \frac{|\text{dohvaeni ciljane klase}|}{|\text{svi dohvaeni}|} $$

# Funkcija koja ocenjuje predloeno pravilo
def oceni_pravilo(ablon, klasa):
    # Ispis predloenog pravila
    print('Pravilo:', ablon, '', klasa, end='\n\n')
    
    # Inicijalno prazan niz klasa dohvaenih instanci
    klase = []
    
    # Dohvatanje svih instanci prema pravilu
    for p in podnorm:
        if ablon in p.newname:
            klase.append(klasa in p.levels)
    
    # Raunanje brojioca i imenilaca mera
    dck = sum(klase)
    sck = sum(klasa in p.levels for p in podnorm)
    sd = len(klase)
    
    # Ispis brojioca i imenilaca mera
    print('Dohvaeni ciljane klase:', dck)
    print('Svi ciljane klase:', sck)
    print('Svi dohvaeni:', sd, end='\n\n')
    
    # Raunanje odziva i preciznosti
    odziv = dck / sck
    preciznost = dck / sd
    
    # Ispis odziva i preciznosti
    print('Odziv pravila:', odziv)
    print('Preciznost pravila:', preciznost, end='\n\n')
    
    # Ispis kontraprimera
    print('Kontraprimeri:')
    
    # Kontraprimer koji smanjuje preciznost
    for p in podnorm:
        if ablon in p.newname and klasa not in p.levels:
            print(p.name, p.levels)
            break
    
    # Kontraprimer koji smanjuje odziv
    for p in podnorm:
        if ablon not in p.newname and klasa in p.levels:
            print(p.name, p.levels)
            break

Primetno je da pravila mogu biti vrlo precizna, pa i postii visoki odziv. Ona, meutim, zahtevaju strunjake koji bi ih pisali i odravali, tako da se uglavnom ne koriste u praksi, osim u posebnim sluajevima ili u kombinaciji sa drugim metodama. Savremeni pristupi zasnovani su na automatskom uenju pravila ili makar njihovoj ugradnji u vektorsku reprezentaciju teksta koji se klasifikuje. Pravila, dakle, mogu biti polazna taka za pravljenje vektora od teksta.

5. Vektorizacija teksta ⮭¶

Ugradnja pravila u vektorsku reprezentaciju teksta moe biti runa, tako to se svojstva (karakteristike, features) izgrauju na osnovu predefinisanih ablona. Prema prethodnim pravilima, mogla bi se napraviti istinitosna svojstva sadri(manager) i sadri(senior) koja bi bila True (1) za imena koja sadre rei (tokene, podniske) manager, odnosno vice, a False (0) za ostale instance. Slino se mogu napraviti i druga runo smiljena vana svojstva (detalji se mogu videti ovde), npr. ona koja pamte prvu i poslednju re ili broj tokena. Ovakve karakteristike uglavnom su diskretne, mada to ne mora biti sluaj.

Predloeni nain vektorizacije teksta gradi takozvane retke reprezentacije. To znai da u njima velika veina karakteristika ima vrednost False (0), a tek poneka True (1). Osim toga, broj svojstava je esto veoma veliki, to oteava obuavanje modela. Ipak, ovakve reprezentacije teksta su vrlo koriene i pokazale su se kao uspene. U praksi, meutim, ne grade se runo, ve se koriste razliiti vektorizatori, koji izgrauju razliite retke reprezentacije. Oni grade kostur reprezentacije na osnovu skupa za obuavanje, pa je neophodno podeliti pripremljeni skup, to je ovde uinjeno kao $33.067 = 24.800 + 8267$.

Poznata vrsta reprezetancije jesu n-gramski modeli. Naime, mogu se posmatrati sve mogue susedne n-torke nekih jedinica (rei, karaktera ili ak bajtova) koji postoje, pri emu svaka n-torka predstavlja jedno svojstvo. Ovakvi modeli daju dobre rezultate u pojedinim primenama, kao to je odreivanje jezika na kom je tekst napisan. Dobra strana im je to to uzimaju u obzir raspored jedinica, ali dobijeni vektori, s druge strane, mogu biti preveliki i previe retki.

N-gramski modeli sa velikom vrednou parametra n i/ili sa sitnim jedinicama (karakterima ili bajtovima) ipak se neto ree koriste. Umesto njih, u upotrebi je vrea rei, koja odgovara unigramskom (n-gramskom za $n = 1$) modelu nad reima. Tekst se u ovom modelu predstavlja vektorom kod koga je svako svojstvo jedna re iz renika, a njegova vrednost zavisi od pojavljivanja te rei u tekstu. Ukoliko se re ne pojavljuje, vrednost vektora na tom mestu je 0 ili False, dok je u suprotnom 1 ili True ili pak taan broj pojavljivanja (nenegativan ceo broj). Ovaj model efektivno zanemaruje poloaj i odnose rei u tekstu. Moe se primetiti da ovaj princip umnogome odgovara ve predstavljenom izdvajanju svojstava na osnovu pravila, odnosno prefedefinisanih ablona.

Upotreba retkih reprezentacija uz metode mainskog uenja odlika je statistikog NLP-ja, kao druge etape razvoja obrade prirodnih jezika. U aktuelnoj, treoj etapi, fokus je na dubokom uenju i gustim reprezentacijama. Za razliku od prethodno razmatranih vektora, koji su izrazito dugaki (na konkretnom primeru: 1895, po jedan atribut za svaku razliitu re u trening skupu), ali retki (na konkretnom primeru: najvie desetak nenula vrednosti), nove reprezentacije imaju znatno manji broj karakteristika realnog tipa, pri emu nijedna nije nula. Osmiljeni su tako da su kosinusno sliniji vektori koji opisuju slinije tekstove.

Veliki je broj gustih modela, kao to su Word2Vec, GloVe, fastText ili ELMo. Oni, meutim, mahom rade nad reima umesto nad sloenim tekstom ili su zahtevni (hardverski i vremenski) za rad. Poznati model koji nije preterano zahtevan, a pritom je u stanju da smisleno vektorizuje ceo tekst jeste BERT. Zasnovan je na transformeru, sloenoj dubokoj neuronskoj mrei sa mehanizmom panje. Koristi se unapred trenirani model, koji se zatim profinjuje (fine-tuning) na konkretnom problemu (ovde klasifikaciji), to je takozvani transfer uenja. Umesto punog modela, najee se koristi neka uproena verzija, kao to je DistilBERT, koji ima 40% manje parametara, 60% je bri, a zadrava ak 97% uspenosti. Ovo uproenje dostupno je kroz modul Sent2Vec.

# Funkcija za sigurno otvaranje fajla
def fopen(fajl, mod):
    # Pravljenje potrebnih direktorijuma
    os.makedirs(os.path.dirname(fajl), exist_ok=True)
    
    # Otvaranje fajla u eljenom modu
    return open(fajl, mod)

# Vektorizacija BERT-om
def bertuj(p):
    try:
        # Pokuaj itanja sa lokala
        with open(f'../bert/{p.newname}.txt', 'r') as f:
            p.bert = eval(f.read())
    
    except:
        # Predvianje nove instance
        bert.run([p.newname])
        
        # Pamenje BERT vektorizacije
        p.bert = [*bert.vectors[-1]]
        
        # Pranjenje niza predvianja
        bert.vectors = []
        
        # Upisivanje na lokal za ubudue
        with fopen(f'../bert/{p.newname}.txt', 'w') as f:
            f.write(str(p.bert))
